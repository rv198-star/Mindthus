结论：当前计划方向基本正确，但“只做校验”会让所有 v3 消息稳定失败，不能恢复处理。下一步应修复字段兼容，同时移除无效重试。

建议修正为：

1. 将根因定性为确定性的 schema 不兼容：`user_id` 与 `userId` 不匹配，不是瞬时超时。
2. 以 `userId` 为规范字段：
   - 从源头把 producer v3 改为发送 `userId`。
   - 若已有在途或存量 v3 消息，在 consumer 边界临时将 `user_id` 映射为 `userId`。
3. 先归一化字段，再执行 schema 校验；缺少两种字段或两者值冲突时，作为不可重试错误拒绝或隔离。
4. 删除该契约错误触发的 timeout retry；重试仅保留给真正的瞬时故障。
5. 验收标准：
   - 现有复现测试由失败转为通过；
   - trace 显示 payload 被正常处理且未进入重试；
   - 补充“字段缺失”和“双字段冲突”两个非重试测试。
6. producer v3 全量修复、存量消息消化后，再移除临时兼容映射。

> 修复 producer v3 的字段名，并在 consumer 边界提供临时版本化兼容；归一化后校验 payload，将 schema 错误标记为不可重试。以现有失败契约测试转绿且运行 trace 不再进入 timeout retry 作为验收条件。
