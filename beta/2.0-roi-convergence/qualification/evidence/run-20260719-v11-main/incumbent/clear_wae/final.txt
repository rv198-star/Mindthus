核心边界：**Workflow 控制顺序、权限和硬门槛；Agent 处理语义不确定性；Evidence 把判断绑定到可验证事实。Agent 永远不能批准自己的建议。**

| 控制层 | 负责内容 | 禁止事项 |
|---|---|---|
| LLM Agent | 理解变更意图；识别影响面、异常组合和遗漏；评估业务风险；解读测试/监控结果；提出灰度、回滚和补证建议 | 修改阈值、豁免规则、把自身结论当证据、直接签发生产放行令 |
| 确定性 Workflow | RBAC 与职责分离；审批人数；步骤顺序；制品锁定；测试/扫描阈值；证据完整性与时效；发布窗口；灰度比例；SLO 自动熔断；超时、回滚和审计留痕 | 判断模糊业务语义，或用“表单已填满”代替真实风险判断 |
| Human Escalation | 不可逆、高爆炸半径、越权、证据冲突、Agent 低置信或分布外场景 | 作为所有普通发布的例行人工节点 |

Agent 输出必须结构化为：`判断 + 证据引用 + 置信度 + 未决问题 + 建议动作`。未决问题不能被摘要隐藏。

**放行所需 Evidence：**

- 与同一 commit、制品 digest、配置版本和目标环境绑定。
- 来自受信系统，带时间戳、签名或不可篡改记录。
- 构建、单测、集成测试、安全/依赖/密钥扫描全部通过。
- 部署清单和配置 diff 已核验。
- 回滚制品、回滚步骤及数据兼容性经过验证。
- 灰度阶段的健康度、错误率、延迟和核心业务指标满足固定阈值。
- 证据仍在有效期内，且不存在未解释的相互冲突。
- 高风险变更额外需要独立审批，以及备份恢复或迁移回退演练证据。

最终规则可压缩为：

```text
ALLOW =
  所有硬门槛通过
  AND 证据完整、可信、新鲜且绑定当前制品
  AND 无超过阈值的未决风险
  AND 所需独立审批已完成
  AND Agent 结论不是唯一放行依据
```

任何证据缺失、过期、冲突或无法关联当前制品，一律 `BLOCK`；Agent 只能请求补证或升级，不能豁免。
