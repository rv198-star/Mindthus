采用 WAE 边界：**Agent 负责处理不确定性；Workflow 负责不可绕过的秩序与权限；Evidence 负责约束结论。**

| 控制层 | 职责 |
|---|---|
| LLM Agent | 解释代码/配置变更、识别语义风险与测试缺口、评估影响面、分析异常、提出灰度与回滚方案；必须输出依据、假设、置信度和未决项 |
| 确定性 Workflow | 身份与 RBAC、职责分离、发布窗口、制品锁定、环境映射、必跑检查、阈值判断、审批人数、灰度节奏、超时/自动回滚、审计留痕 |
| 人工审批 | 高影响、不可逆、跨权限、证据冲突及例外放行；Agent 无权授予豁免 |

放行所需 evidence：

- **制品一致性**：测试对象与待发布制品的 digest 完全一致，来源和构建链可追溯。
- **质量验证**：规定范围的单元、集成、端到端、回归测试通过；覆盖范围与变更影响匹配。
- **安全合规**：漏洞、密钥、依赖、许可证和策略扫描通过确定性阈值。
- **运行准备**：监控、告警、容量、迁移方案、灰度方案和可执行回滚方案已验证。
- **审批有效性**：审批人有权、职责分离满足、审批绑定当前 commit/digest，且未过期。
- **证据完整性**：证据来自可信系统，带时间、范围、结果和不可篡改引用；缺失、过期或相互矛盾即阻断。

最终规则：

```text
放行 =
全部硬门禁通过
AND evidence 有效且绑定同一制品
AND 无未解决 blocker
AND 满足风险等级对应的人工审批
```

Agent 的“建议放行”、自评分或自然语言总结**不能作为放行证据，也不能覆盖硬门禁**。任何例外必须由确定性流程记录授权人、范围、补偿措施和失效时间。
