# H2.1 — MPG↔SELA Companion-gate Causal Repair

Status: `STATIC_QUALIFIED / LIVE_NOT_STARTED`

## Claim under test

The clean H2 continuation proved that the frozen candidate can execute
`entry -> index -> MPG`, but it then read SELA on a prompt that had no
system-efficiency/trend-versus-local-advantage conflict. H2.1 tests the narrower causal
claim: the topology may still qualify if that extra read is caused by an over-broad
MPG companion instruction and one subtractive gate correction removes the false load
without suppressing a genuine SELA+MPG handshake.

This Mission must not reinterpret the prior failure, relax owner fidelity, or claim
that a useful final answer cancels a lifecycle failure.

## Authorized correction

Replace the broad instruction “when a system-efficiency/trend-based mainline appears”
with a conjunctive gate:

1. the user's stated decision context contains a system-efficiency or long-term trend
   claim; and
2. it also contains a real local advantage that conflicts with that claim.

A fixed operational target, deadline, carrier, exposure, or path volatility alone must
not satisfy the companion gate. When both conditions are present and MPG owns the
path-carrying action, MPG must still read SELA as a supporting owner before answering.

The correction may update matching declarations in the MPG owner root and its detailed
methodology resource so the contract is internally consistent. This is one conceptual
correction, not two iteration opportunities. No other owner method, trigger, guardrail,
output contract, or support resource may change.

After the first live call, correction budget is zero.

## Three causal probes

lifecycle evidence, and a 275,000 Generator rejection line inside the 400,000 outer

### C1 — MPG-only regression and qualification Q1/Q2

Use the frozen H2 Q1 prompt, then resume the exact thread with frozen H2 Q2. This pair
also serves as H2.1 qualification Q1/Q2.

Pass Q1: entry only, no index/owner or direction, one answer-changing blocking question.

Pass Q2: `entry -> index -> MPG`, no SELA or other owner, useful path-carrying action,
and no user-visible routing round.

### C2 — genuine SELA+MPG positive

Use a natural decision that explicitly contains a long-term system-efficiency/trend
claim, a conflicting real local advantage, a fixed actor/mainline, concrete carrier,
meaningful exposure, path volatility, and a current path choice.

Pass: `entry -> index -> MPG -> SELA`, no other owner, one visible action thesis, and
SELA supports direction while MPG owns the path action.

### C3 — companion near-negative

Use a natural MPG decision that mentions a long-term trend or system-efficiency claim
but explicitly states no conflicting local advantage. Keep actor, mainline, carrier,
exposure, volatility, and current path choice complete.

Pass: `entry -> index -> MPG`, no SELA or other owner, useful action, no routing round.

C2 and C3 prompts are byte-identical except for the single result word
`superior` versus `equivalent`. Any other prompt delta invalidates the causal pair.

Prompts and criteria freeze before C1. Any causal semantic failure rejects H2.1. Do not
patch and rerun.
