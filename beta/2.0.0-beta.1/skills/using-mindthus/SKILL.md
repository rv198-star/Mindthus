# using-mindthus — Arbitration Only (2.0 Beta)

## Core

Arbitrate unresolved ownership. Do not act as a mandatory pre-router and do not repeat
the Passive Activation Kernel.

## Mainline

1. Exit to direct execution when no hard judgment remains.
2. Exit to evidence acquisition when missing facts or runtime proof block judgment.
3. Transfer immediately when one concrete owner is already clear.
4. When multiple owners remain plausible, identify the judgment object and choose the
   owner whose decision would change the problem, evidence bar, action, or stop.
5. Preserve all active passive obligations during transfer. One execution owner does
   not imply one active primitive.

Use these axes only to break a real tie:

| Judgment object | Owner |
| --- | --- |
| unclear problem shape or executable breakdown | `mindthus-beta:3l5s` |
| malformed proposition, false binary, or structural ambiguity | `mindthus-beta:edsp` |
| system-efficiency direction versus local advantage | `mindthus-beta:sela` |
| carrier, exposure, path volatility, or commitment action | `mindthus-beta:mpg` |
| agent/workflow/evidence controller mismatch | `mindthus-beta:wae` |
| bounded artifact with a named value-gain target | `mindthus-beta:tvg` |
| durable Mission state, authority, recovery, or closure | `mindthus-beta:tplan` |

If no owner stabilizes, ask at most one decision-changing clarification. When the
bounded Kernel action is already clear, apply it directly without invoking another
method. Do not load every method for comparison.

## Handoff

Keep this internal state when useful:

`execution_owner / thesis_owner / primitive_obligations[] / handled[] / remaining[]`

Expose the structure only on request or when a handoff fails. Stop after transfer; the
selected owner performs the full judgment.

## Boundary

Do not use this skill merely because Mindthus may apply. Do not route by model name,
keywords, or artifact type. Clear owner means direct Beta owner invocation. If Stable
and Beta skills are both active, stop and report an isolation conflict.
