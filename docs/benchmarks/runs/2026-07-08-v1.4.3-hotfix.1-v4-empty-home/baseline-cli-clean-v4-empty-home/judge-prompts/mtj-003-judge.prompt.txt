**SSR 迁移步骤**

2. **确定 SSR 技术方案**
   - 如果是 React：可考虑 Next.js、Remix，或现有 Node 服务加 React SSR。
   - 如果是 Vue：可考虑 Nuxt。
   - 如果后端已有模板体系，也可以回到后端模板 + 局部前端增强。
   - 明确是否需要同构渲染、流式渲染、边缘渲染、静态生成。

3. **设计新的路由结构**
   - 将前端路由映射为服务端路由。
   - 明确每个 URL 对应的页面组件、数据来源、缓存策略和权限要求。
   - 保持原 URL 尽量不变，减少 SEO 和外链损失。

4. **迁移数据获取逻辑**
   - 把首屏必须的数据请求从浏览器端移到服务端。
   - 梳理接口认证、Cookie、Session、Header 透传。
   - 避免服务端渲染时重复请求、瀑布请求和慢接口拖垮首屏。

5. **改造登录态和权限**
   - 从 localStorage/token 模式迁移到更适合 SSR 的 Cookie 或 Session 方案。

   - 处理 401、403、重定向、续期等逻辑。

6. **组件兼容 SSR**
   - 清理直接访问 `window`、`document`、`localStorage` 的代码。
   - 浏览器专属逻辑放到客户端生命周期中执行。
