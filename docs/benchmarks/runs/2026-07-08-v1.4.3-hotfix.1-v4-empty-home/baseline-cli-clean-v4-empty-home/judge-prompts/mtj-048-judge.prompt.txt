Prompt 工程里确实有一部分像“话术”：怎么表述目标、限定范围、给出格式、减少歧义。但它的本质更接近**人机协作的任务设计**，不只是把话说漂亮。

更准确地说，Prompt 工程包括：

- **任务拆解**：把模糊需求变成模型能执行的步骤。
- **上下文组织**：提供必要背景、数据、约束和示例。
- **输出控制**：规定格式、风格、边界、禁止项。
- **误差管理**：减少幻觉、偏题、遗漏和不稳定输出。
- **评估迭代**：根据结果不断调整提示词、流程或工具调用。

如果只是日常问答，确实像“提问技巧”。  
但在实际产品、自动化流程、Agent、客服、代码生成、数据分析里，Prompt 工程更像是**用自然语言写接口和流程规范**。

所以可以说：

> 低阶 Prompt 工程像话术；高阶 Prompt 工程是任务建模、上下文工程和系统设计。
