可以。建议把第三个 fallback 放在现有两个分支之后，作为最后兜底，原则是：

1. 不再依赖不稳定测试路径里的外部状态
2. 返回一个确定性的默认结果
3. 只捕获预期异常，避免吞掉真实 bug
4. 加日志或埋点，方便后续确认是否频繁走到兜底

大致结构可以是：

```ts
try {
  return await primaryFlow()
} catch (err) {
  try {
    return await firstFallback()
  } catch (err) {
    try {
      return await secondFallback()
    } catch (err) {
      return stableDefaultFallback()
    }
  }
}
```

如果现在已有两个 fallback，我会加成：

```ts
return stableDefaultFallback()
```

并补一个测试用例覆盖“主逻辑 + 两个 fallback 都失败时，仍返回稳定结果”的场景。
