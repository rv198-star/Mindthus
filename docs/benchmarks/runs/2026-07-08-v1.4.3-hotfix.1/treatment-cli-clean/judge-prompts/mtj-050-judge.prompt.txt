- **话术技巧**：怎么说得更容易让模型配合，比如“请一步步分析”“用表格输出”。
- **Prompt 工程**：设计一套让模型稳定完成任务的输入结构，包括目标、角色、背景、数据、边界、输出格式、错误处理、示例、评估标准等。

比如：

普通话术：

> 帮我写一篇产品介绍，要吸引人。

更工程化的 Prompt：

> 你是 B2B SaaS 文案顾问。请基于以下产品信息，面向中小企业 CEO，写一段 150 字以内的官网首屏文案。要求突出降本增效，避免夸张承诺，语气专业克制，输出包括标题、副标题、CTA 三部分。

后者不是单纯“会说话”，而是把任务条件结构化，让结果更可控。

2. **任务建模层**
   明确模型要扮演什么角色、输入是什么、输出标准是什么、哪些约束不能违反。  
   例如不是说“帮我写文案”，而是定义受众、渠道、转化目标、品牌语气、禁用表达、评价标准。

3. **上下文组织层**
   给模型足够但不过载的信息：背景、样例、反例、优先级、边界条件。很多效果差不是因为“话术不够高级”，而是上下文缺失或冲突。

4. **流程设计层**
   复杂任务通常不是一个 prompt 解决，而是拆成：理解需求、生成方案、校验、修改、输出。这里更像工作流设计。

5. **评估与迭代层**
   好 prompt 不是“看起来会说”，而是能稳定地产生可用结果。需要测试不同输入下是否可靠。

所以，如果只是让模型“说得好听一点”，那是话术；但真正的 Prompt 工程更像是**需求分析 + 任务设计 + 约束表达 + 质量控制**。
