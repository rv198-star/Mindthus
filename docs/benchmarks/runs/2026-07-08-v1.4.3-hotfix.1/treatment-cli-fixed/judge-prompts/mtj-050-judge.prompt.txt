2. **上下文选择**
   给模型哪些信息，不给哪些信息，信息之间的优先级是什么。

3. **约束设计**
   比如风格、格式、边界、禁止事项、证据要求、推理深度。

4. **输出校准**
   让结果可用、可检查、可复用，而不是只看起来顺滑。

5. **工作流控制**
   有些复杂任务不是一个 Prompt 解决，而是拆成提问、生成、审查、修正、验证几个步骤。

所以，**话术是表层，任务建模才是本质**。

一个简单判断标准是：  
如果换个说法结果就好一点，那是话术；  
如果你改变了模型理解问题、使用信息、判断取舍和交付结果的方式，那就是 Prompt 工程。
