**内容结构**

售后规则页建议按用户问题组织，而不是按内部规则组织：

1. **我想退款**
   - 哪些订单可退款
   - 退款金额怎么算
   - 优惠券、积分、运费是否退
   - 赠品是否需要退回
   - 退款到账时间

2. **我想换货**
   - 哪些商品支持换货
   - 换货时效
   - 是否需要补差价
   - 运费承担规则
   - 库存不足怎么办

3. **订单有赠品**
   - 退款时赠品怎么处理
   - 换货时赠品是否保留
   - 赠品损坏 / 缺失怎么办
   - 赠品价值是否影响退款

4. **特殊情况**
   - 部分退款
   - 组合商品
   - 满减 / 套餐 / 加价购
   - 已发货、已签收、已拆封
   - 售后审核失败原因

**交互方式**

建议做成“规则问答 + 操作引导”：

用户先选场景：

`我要退款` / `我要换货` / `订单含赠品` / `不知道选哪个`

然后继续选择：

`已发货` / `已签收` / `未发货`  
`商品完好` / `商品损坏`  
`赠品已收到` / `赠品未收到`

最后给出明确结果：

- 是否可申请
- 需要准备什么
- 预计处理时间
- 下一步按钮：`去申请退款`、`去申请换货`、`联系客服`

- 后台可维护文案、条件、跳转动作
