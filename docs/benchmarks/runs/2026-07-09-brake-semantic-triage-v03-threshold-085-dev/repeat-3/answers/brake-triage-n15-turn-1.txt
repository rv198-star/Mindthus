建议不要直接全量前移，而是做一轮小流量验证，避免过早提醒导致遗忘或反感。

方案：

1. **提醒节奏**
   - 保留第三轮效果最好的“日历邀请”作为主形式。
   - 新增更早提醒：课前 **48小时** 发日历邀请/确认提醒。
   - 保留临近提醒：课前 **2小时** 发简短提醒。
   - 如课程在早上，可加课前一天晚上 **20:00** 的轻提醒。

2. **消息内容**
   - 继续保持短正文，只强调三件事：
     - 课程时间
     - 入口/地点
     - 一键确认或加入日历
   - 示例：
     > 明天 19:30 开课，已为你发送日历邀请。点击确认后，系统会在开课前提醒你。

3. **实验设计**
   - A组：维持当前日历邀请方案。
   - B组：日历邀请时间前移到课前48小时。
   - C组：课前48小时日历邀请 + 课前2小时短提醒。
   - 每组人数尽量接近，至少跑一轮完整课程周期。

4. **目标拆解**
   - 当前 82%，目标 85%，只需要提升 3 个百分点。
   - 优先看：
     - 日历邀请接受率
     - 课前2小时提醒点击率
     - 实际到课率
   - 如果 B 组提升不明显，说明“单纯前移”价值有限；如果 C 组提升明显，说明关键是“提前锁定 + 临近唤醒”。

5. **推荐执行**
   - 今天先上 C 组小流量，比例 20%-30%。
   - 若到课率达到或超过 85%，下一轮扩大到 70%-100%。
   - 若只到 83%-84%，再增加“确认后权益”或“缺席影响提示”，比如资料优先发放、席位保留提醒等。

最推荐的版本是：**课前48小时日历邀请 + 课前2小时极短提醒**。这比单纯把提醒时间前移更有机会从 82% 拉到 85%。
