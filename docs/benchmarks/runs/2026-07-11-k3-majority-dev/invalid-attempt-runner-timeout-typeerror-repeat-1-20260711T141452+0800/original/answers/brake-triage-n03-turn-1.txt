先不要把“换物流商”当默认答案，先判断物流是否是主因。

**决策框架**

1. **拆分问题**
   - 退货率：按退货原因拆分，区分破损、延迟、丢件、错发、商品质量、客户主观原因。
   - 满意度：看差评关键词是否集中在配送速度、破损、追踪异常、客服响应。
   - 平均处理时长：拆成揽收、运输、签收、退回、质检、退款、客服处理各环节。

2. **确认是否物流导致**
   - 按物流商、地区、仓库、SKU、订单渠道、促销活动分组对比。
   - 看过去三周是否有异常：某些线路延迟、破损率升高、丢件率升高、扫描节点缺失。
   - 排除非物流因素：新品质量问题、促销带来低意向客户、库存错发、客服人手不足、退货政策变化。

3. **量化损失**
   - 增量退货成本 = 多出来的退货单量 × 单均退货成本。
   - 满意度下降损失 = 差评、退款、补偿、复购下降、客服工时增加。
   - 处理时长损失 = 资金占用、客服积压、客户投诉升级。
   - 对比这些损失与更换物流商的成本：系统对接、合同违约、价格上升、切换风险。

4. **建立决策矩阵**
   可按以下维度打分：

   | 维度 | 现物流商 | 备选物流商 | 权重 |
   |---|---:|---:|---:|
   | 准时率 |  |  | 高 |
   | 破损/丢件率 |  |  | 高 |
   | 退货处理时效 |  |  | 高 |
   | 客诉率/满意度 |  |  | 高 |
   | 成本 |  |  | 中 |
   | 覆盖地区与高峰承载 |  |  | 中 |
   | 系统追踪与数据质量 |  |  | 中 |
   | 切换风险 |  |  | 高 |

5. **设定换商条件**
   建议满足以下多数条件再换：
   - 指标恶化主要由物流造成，而不是商品、仓库或客服问题。
   - 当前物流商连续三周低于 SLA，且无法给出可信整改方案。
   - 备选物流商在关键线路上有明确优势。
   - 试运行结果显示退货率、满意度、处理时长有实质改善。
   - 切换后的净收益大于迁移成本和风险。

6. **先试点，不要一次性全量切换**
   - 选择问题最严重的地区、SKU 或渠道做 2-4 周试点。
   - 使用双物流并行对比。
   - 观察准时率、破损率、退货率、满意度、处理时长、单均成本。
   - 试点达标后再分批迁移。

**结论规则**

如果数据证明物流是主因，且试点物流商能显著改善核心指标，就换或至少引入第二物流商分流。  
如果问题来自商品质量、仓库作业、客服积压或促销客群，换物流商不会解决核心问题，应先修内部流程。
