可以把方向从“给每个页面补丁”改成“统一售后规则入口 + 场景化判断”。

**方案核心**

建立一个「售后规则中心」，所有退款、换货、赠品、运费、时效、特殊商品规则都从这里维护和展示，退款页、换货页、赠品页不再各自写死提示，只引用同一套规则结论。

**入口设计**

1. **订单详情页放主入口**
   - 按订单状态展示：`申请售后` / `查看售后规则`。
   - 带入订单、商品、支付、赠品、活动信息，直接给用户判断结果。

2. **售后申请页做规则判断**
   - 用户选择：退款、换货、补发、赠品问题。
   - 系统返回：是否可申请、截止时间、需要材料、费用承担、下一步按钮。
   - 不再用大段说明堆页面。

3. **帮助中心保留完整规则页**
   - 给用户和客服查全量规则。
   - 支持搜索：退款、换货、赠品、运费、质量问题、超时等。

**页面结构**

售后规则入口首页建议分三块：

- **智能判断**：选择订单或输入商品，判断当前是否能退换。
- **常见场景**：退款、换货、赠品、少发漏发、质量问题、超时申请。
- **完整规则**：按商品类型、活动类型、售后类型查看。

每条规则详情统一用这个结构：

- 适用范围
- 可申请条件
- 不支持情形
- 申请时限
- 运费/差价/赠品处理
- 需要提交的材料
- 操作入口

**后台能力**

要做一个简单的规则配置，不要继续靠页面文案维护。

规则字段可以包括：

- 售后类型：退款 / 换货 / 补发 / 赠品
- 商品类型：普通商品 / 定制商品 / 生鲜 / 虚拟商品等
- 订单状态
- 签收时间范围
- 是否参与活动
- 是否含赠品
- 是否支持退换
- 运费承担方
- 生效时间与版本
- 前台展示文案

**迁移步骤**

1. 先梳理现有退款页、换货页、赠品页所有补丁文案。
2. 合并成一套规则表，去掉重复和冲突内容。
3. 上线统一售后规则入口。
4. 三个旧页面只保留轻提示和跳转，不再单独维护复杂说明。
5. 客服话术、帮助中心、前台页面统一引用同一套规则。

**上线节奏**

第一期建议先做轻量版：

- 订单详情入口
- 售后规则首页
- 退款 / 换货 / 赠品三类规则详情
- 后台可配置文案和生效状态

第二期再做：

- 根据订单自动判断是否可售后
- 按活动、赠品、商品类型自动展示差异规则
- 客服侧同步规则结论

目标是把“页面补说明”变成“规则一次配置，多处复用”。这样后面改售后政策时，不用再分别改退款页、换货页、赠品页。
