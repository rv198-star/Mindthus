# 3L5S / 三层五步

## 这是什么

3L5S 是一个通用问题处理内核，意思是 `Three Layers + Five Steps`。

三层是：

- `Discovery`
- `Definition`
- `Resolution`

五步是：

- `Baseline`
- `Target`
- `Gap`
- `Strategy`
- `Breakdown`

## 解决什么问题

3L5S 主要处理两类问题：

1. 现象很多，但真正的问题还不清楚
2. 问题已经清楚，但太大，没法直接执行

它的作用是把“看起来像问题的东西”收敛成可复述、可定位、可证伪、可执行的对象。

## 核心判断

先分清两件事：

- 现在到底发生了什么
- 应该把问题定义成什么

如果问题还不清楚，就先做 `Discovery -> Definition`。  
如果问题已经清楚但太大，就进入 `Resolution`，再用 `Baseline -> Target -> Gap -> Strategy -> Breakdown` 落地。

## 怎么用

常见路径有两条：

- `Single-layer BTGSB`：问题已接受，直接拆执行
- `Three-layer 3L5S`：信号、问题、行动还混在一起，先分层

最重要的不是把层数跑满，而是不要把观察、定义、执行混成一句话。

## 常见误用

- 把 3L5S 当固定模板
- 还没定义问题就开始做方案
- 把完整结构误当成正确判断

## 边界

3L5S 不替代领域证据、运行时验证或 stakeholder 判断。

如果问题已经非常明确，别为了形式跑完整三层。

## 与其他方法的关系

- `EDSP` 处理 3L5S 里模糊的结构选择
- `WAE` 处理 3L5S 步骤的控制权边界
- `tvg` 用在 3L5S 产物已经成型但还偏薄时
