# EDSP / Extreme Deduction + Scenario Projection

## 这是什么

EDSP 是一个结构判断方法，意思是 `Extreme Deduction + Scenario Projection`。

它先做极限推演，把模糊问题压成有限的结构坐标系，再把具体场景投进去看边界。

## 解决什么问题

EDSP 适合处理这类判断：

- A 和 B 都像对，但到底哪个更对
- 这是不是个真的二选一
- 命题本身是不是有问题
- 某个原则在具体场景里该怎么落地

## 核心判断

先做 `Extreme Deduction`：

- 拆变量
- 推极端
- 合并成少量离散结果
- 看现实漂移方向

如果坐标系稳定，再做 `Scenario Projection`。

关键不是把答案讲得漂亮，而是先弄清楚原问题是不是结构上成立。

## 怎么用

三步就够：

1. 看结构是不是模糊
2. 把关键变量推到极端
3. 在稳定坐标系里做场景投影

如果结构都还不稳，不要急着上场景。

## 常见误用

- 把 EDSP 当证据替代品
- 先做场景，再回头补结构
- 用漂亮的坐标系掩盖命题本身不成立

## 边界

EDSP 不替代领域研究、运行时证明或 stakeholder 判断。

如果真正缺的是事实，不是结构，就先去补事实。

## 与其他方法的关系

- `3L5S` 提供问题内核
- `WAE` 约束谁控制判断、谁控制证据
- `tplan` 可以把 EDSP 变成 Mission 里的选择钩子
