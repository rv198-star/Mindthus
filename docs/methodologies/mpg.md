# MPG / Mainline-Path Game

## 这是什么

MPG 是 `Mainline-Path Game / 主线-路径博弈`，用来处理一个很容易被忽视的问题：

> 看对主线，不等于穿过路径。

SELA 可以帮我们看清长期主线，比如系统效率正在压过局部优势，某个大趋势正在形成，或者一个组织必须转型。但真实行动里，主线不会直线兑现。它会经过政策摇摆、地方激励、市场回撤、现金流压力、组织阻力、技术迁移成本和叙事波动。

MPG 关心的是：这个行动者用什么载体、什么节奏、什么暴露度和什么触发条件，才能把主线穿过路径波动，而不是死在主线兑现之前。

它的核心交付不是“风险很高 / 机会很大”这样的判断，而是：

> Path-Carrying Strategy / 主线承载方案

## 解决什么问题

MPG 解决的是“主线正确但路径会杀死承载者”的问题。

典型场景包括：

- AI 长期会走出来，但某只股票、某个仓位或某家创业公司可能经历多次大回撤。
- 房价长期承压，但中间可能有政策托底、信贷放松和局部反弹。
- 中央方向清楚，但地方有财政、就业、稳定和执行激励。
- 公司必须转型，但老业务现金流、销售激励和客户迁移会反复拉扯。
- 一个人长期可能成功，但他当前所在方向、公司或现金流载体可能先破产。
- 技术路线长期更优，但迁移过程会撞上 legacy、合规、可用性和团队能力。

这些问题里，简单定性会太松，精确数字又常常是假装确定。MPG 允许用非精准量化显影来呈现变量关系，但数字是假设，关系才是重点。

## 核心判断

MPG 的核心判断是：主趋势决定初始方向，对抗力量塑造并可能改写路径。

一个主线要进入 MPG，不能只是裸命题。它必须被限定成可行动命题：

- 谁的路径？
- 多长时间内？
- 通过什么载体？
- 需要承受什么波动？
- 什么条件下算主线兑现？
- 什么条件下要下调、换载体或退出？

所以 MPG 的输出必须包含这些字段：

- `qualified_mainline`
- `carrier_vehicle`
- `counter_force_map`
- `exposure_budget`
- `optionality_design`
- `action_posture`
- `trigger_conditions`
- `mainline_challenge`

如果这些字段没有改变行动姿态、暴露度、载体选择、触发条件或证据要求，那就还不是合格的 Path-Carrying Strategy / 主线承载方案。

## 怎么用

MPG 的使用流程很短：

1. 限定主线：把“AI 会赢”“房价会跌”“公司要转型”改成带行动者、期限、条件和失败含义的命题。
2. 画出对抗力量：不是把它们当坏人，而是看它们如何弯曲路径。
3. 拆开主线、承载者和载体：主线对，不等于当前股票、公司、团队、政策机制或技术方案能活到终点。
4. 设暴露预算：钱、时间、现金流、士气、权威、信任、流动性都可能是预算。
5. 设计可选性：分阶段、换载体、保留现金、可逆迁移、小规模试探、双轨过渡。
6. 选行动姿态：`commit`、`stage`、`hedge`、`wait`、`switch vehicle`、`probe`、`hold` 或 `exit`。
7. 写触发条件：什么情况下加仓、减仓、换载体、暂停、退出或反过来质疑主线。

最重要的一步是最后的 `mainline_challenge`。极端情况下，MPG 可以推翻被限定后的 SELA 主线：比如“人总会死”是真的，但“这个人十年内大概率死亡”不一定成立；“房价长期承压”可能是真的，但“当前高杠杆空头能撑过未来 12 个月”可能是假的。

## 具体案例

### 案例 A：AI 股票长期主线和 70% 回撤

裸主线是“AI 一定会走出来”。MPG 不会停在这里，而是改成：某个投资者是否能用当前仓位穿过未来 3-5 年可能出现的多轮估值回撤。

对抗力量包括利率、估值过高、监管、资本开支周期、竞争格局变化和叙事退潮。承载者不是“AI”，而是投资者的现金流、心理承受、仓位结构和再平衡纪律。

可能的主线承载方案是：主线接受，但不接受单一高波动载体；采取分阶段配置，保留现金预算，设定回撤再平衡触发条件，并把“AI 长期增长”与“这只股票必须一直赢”拆开。

### 案例 B：中央方向清楚，但地方路径会波动

中央可能长期希望降杠杆、压地产依赖或推进某种产业结构调整。但地方政府面对财政、就业、土地收入和短期稳定压力，可能会阶段性放松、拖延、变形执行。

MPG 不把地方力量当成“错误”，而是把它看作路径塑形力量。承载方案可能不是“中央对，所以地方所有反弹都无效”，而是区分长期方向、地方缓冲、政策容忍边界和触发条件。

有效输出会告诉行动者：哪些地方反弹只是路径波动，哪些反弹意味着中央约束被削弱，哪些指标需要重新评估主线。

### 案例 C：公司必须转型，但现金流会先杀人

一家公司长期必须从旧业务转到新业务。SELA 可能支持这个方向，因为新系统效率更高。但 MPG 会继续问：旧业务现金流能撑多久？销售团队是否愿意卖新产品？客户迁移会不会导致短期收入断崖？组织权威是否足够？

主线承载方案可能是双轨 KPI、分客户层迁移、设现金流红线、先做小规模可复用模块，而不是一句“方向正确，全力转型”。

### 案例 D：一个人长期会成功，但当前载体可能失败

某个人的能力、韧性和方向感可能支持长期成功。但他当前创业项目、行业窗口、现金流或合伙关系可能无法承载这个主线。

MPG 的价值是把“这个人会成功”和“当前公司会成功”拆开。承载方案可能是保留个人能力主线，降低当前项目暴露，设计失败后的再进入路径，而不是把所有资源压在一个脆弱载体上。

## 常见误用

第一种误用，是把 MPG 当成更复杂的风险清单。风险清单只列危险，MPG 必须改变承载方式。

第二种误用，是没有主线就使用 MPG。如果主线本身还没判断清楚，先回到 SELA 或 EDSP。

第三种误用，是没有承载者。没有行动者、载体、暴露预算或触发条件时，MPG 会退化成泛泛而谈。

第四种误用，是把对抗力量都当成坏东西。很多对抗力量是局部理性，比如地方财政压力、旧业务现金流、团队迁移成本和市场估值修正。

第五种误用，是只追求活下来。活下来但彻底放弃主线、损毁核心优势或锁死未来，也不是 MPG 成功。

## 边界

MPG 不替代 SELA。SELA 看整体趋势与局部优势的关系；MPG 负责主线承载方案。

MPG 不替代 EDSP。如果命题本身是伪二选一、结构边界不清或主线无法限定，先用 EDSP。

MPG 不替代 WAE。如果真正的问题是 workflow、agent、evidence 或人的控制权边界，应该交给 WAE。

MPG 不替代事实研究。市场数据、医疗判断、法律风险、政策事实和财务建议都必须受 Evidence / Claim Ceiling 约束。

## 与其他方法的关系

- `SELA`：看清主线和整体/局部关系。MPG 接住 SELA 的方向，并检查路径是否能穿过。
- `EDSP`：当主线还不成形，或者命题本身有结构问题时，先用 EDSP。
- `WAE`：当路径问题其实是控制权问题时，WAE 主导，MPG 只处理承载策略。
- `3L5S`：当问题还没定义清楚时，先用 3L5S。
- `TVG`：当 MPG 输出的方案看起来完整但交付价值仍浅时，用 TVG 加深。
- `tplan`：长任务里由 tplan 记录状态、证据和决策钩子，遇到主线承载问题时再路由到 MPG。
