# Entry Triage / 入口分诊

Entry Triage is a before-route primitive. It decides whether the user has presented a
hard judgment point before any Mindthus method owns the answer.

It is not a keyword router. It reads semantic shape: what is trying to define the
object, what evidence is missing, and which mistake would change the next action.

## Disease Families

Use Entry Triage when one of these high-frequency shapes appears:

- definition authority contest
- single-factor attribution
- local repair spiral
- forced binary prediction
- no-data numeric comparison
- trend-driven migration

These families are wake-up candidates, not automatic method calls. A clear, low-risk,
fact-sufficient task still stays direct.

## Trigger Register

Representative v3 audit triggers:

- green tests imply release readiness -> Input Framing Audit
- X is just Y, therefore impossible -> Whole Elephant Protocol
- business success reduced to one factor -> Whole Elephant Protocol
- third prompt rule or third fallback -> Anti-Spiral
- who is right plus active buy or decide context -> Decision Context Calibration
- no-data numeric comparison -> AQM evidence ceiling
- trend-driven migration -> Input Framing Audit, then SELA or MPG only if the object fits

Every new trigger must pass public negative and shadow controls. If a change improves
positive wake-up but regresses negative controls, it fails.

## Ownership Tie-Breaks

EDSP owns bare A/B structural choice. SELA owns local expertise versus automation
scale. If both appear, EDSP exposes the axis before SELA checks system efficiency; do
not split the final answer into half EDSP and half SELA.

AQM stays inside the active judgment owner. Hypothetical numbers must not decide, and
invented arithmetic must not become the decision basis.

## Output Gates

Root-cause evidence gate: timeline and metrics before conclusion. Missing
evidence means the root-cause section is hypothesis, not fact.

Anti-Spiral hard brake: same local repair count >= 3 must stop adding and move
upstream or equal-replace; subtraction is allowed.

Visible consequence probe: if a wrong definition would redirect optimization, name
that consequence explicitly before advice.

## Boundary

Entry Triage only changes the route, evidence requirement, stopping condition, or first
answer shape. If it changes none of these, skip it.
