# Entry Triage / 入口分诊

Entry Triage is a before-route primitive. It decides whether the user has presented a
hard judgment point before any Mindthus method owns the answer.

It is not a keyword router. It reads semantic shape: what is trying to define the
object, what evidence is missing, and which mistake would change the next action.

## Disease Families

Use Entry Triage when one of these high-frequency shapes appears:

- definition authority contest
- single-factor attribution
- local repair spiral
- forced binary prediction
- no-data numeric comparison
- trend-driven migration

These families are wake-up candidates, not automatic method calls. A clear, low-risk,
fact-sufficient task still stays direct.

## Trigger Register

Representative v3 audit triggers:

- green tests imply release readiness -> Input Framing Audit
- X is just Y, therefore impossible -> Whole Elephant Protocol
- business success reduced to one factor -> Whole Elephant Protocol
- third prompt rule or third fallback -> Anti-Spiral
- who is right plus active buy or decide context -> Decision Context Calibration
- no-data numeric comparison -> AQM evidence ceiling
- trend-driven migration -> Input Framing Audit, then SELA or MPG only if the object fits

Every new trigger must pass public negative and shadow controls. If a change improves
positive wake-up but regresses negative controls, it fails.

## V5 Target Trigger Register

These anchors come from the V4 no-load cases. They are semantic shapes, not keyword
rules, and only fire when the next action would change.

V5/#104 no-load stabilization register:

- authority or tenure used as root-cause proof plus requested incident write-up -> Input Framing Audit
- trend label used as migration mandate -> Input Framing Audit
- local green signal treated as release authorization -> Input Framing Audit
- business/store success reduced to one product attribute before copywriting -> Whole Elephant Protocol
- forced yes/no replacement prediction over role/task family -> EDSP structural judgment
- third local prompt rule after two failed edits -> Anti-Spiral hard brake
- third fallback branch after two fallbacks or unstable tests => Anti-Spiral hard brake
- definition/essence reduction to communication/tactic only -> Formal Answer Gate / Expression Discipline
- no measured data plus concrete numeric risk comparison -> AQM evidence ceiling

V5 target triggers: no-load anchors from public V4: operator-expertise root cause;
trend-slogan migration; green-tests release; ingredient/metric/interface explains
business result; bare yes/no replacement; third prompt rule/fallback/local patch;
prompt-engineering/skill/script essence; no-data numeric comparison. Trigger only when
route/evidence/stop/first-sentence action changes; low-risk fact-sufficient tasks stay
direct.

| V5 case family | Route owner | Mechanical gate | Negative boundary |
| --- | --- | --- | --- |
| operator expertise asserts a root cause | Input Framing Audit | root-cause evidence gate: timeline, metrics, traces, or mark root cause as hypothesis | ordinary incident writing with provided evidence stays direct |
| trend slogan demands migration | Input Framing Audit | ask target function before migration steps | explicit migration decision with complete constraints can execute |
| green tests imply release now | Input Framing Audit | release-readiness gate: rollback, gray, monitoring, user impact, risk acceptance | drafting a status note after release authority is settled stays direct |
| business success reduced to one ingredient, metric, or interface | Whole Elephant Protocol | reconstruct target job, value carrier, and result controller before copy or strategy | narrow marketing copy with declared scope may stay bounded |
| bare yes/no replacement or role-erasure prediction | EDSP | test both extremes before giving the practical branch | low-risk factual yes/no stays direct |
| third prompt rule, third fallback, or next local patch after instability | Anti-Spiral | stop adding; move upstream, subtract, or equal-replace | normal debugging with new evidence delta can continue |
| essence claim about prompt engineering, skills, scripts, or local carrier | Expression Discipline plus Whole Elephant | first sentence must name definition owner before local truth | user-owned terminology can be marked as local convention |
| no-data numeric comparison | AQM evidence ceiling | numbers may expose relationships only as assumptions; no computed verdict | measured data or user-provided inputs can be calculated normally |

## Ownership Tie-Breaks

EDSP owns bare A/B structural choice. SELA owns local expertise versus automation
scale. If both appear, EDSP exposes the axis before SELA checks system efficiency; do
not split the final answer into half EDSP and half SELA.

AQM stays inside the active judgment owner. Hypothetical numbers must not decide, and
invented arithmetic must not become the decision basis.

## Output Gates

Root-cause evidence gate: timeline and metrics before conclusion. Missing
evidence means the root-cause section is hypothesis, not fact.

Anti-Spiral hard brake: same local repair count >= 3 must stop adding and move
upstream or equal-replace; subtraction is allowed.

Visible consequence probe: if a wrong definition would redirect optimization, name
that consequence explicitly before advice.

Loaded-action probes:

- Input Framing Audit must withhold root-cause/release authority until evidence,
  rollback, gray rollout, monitoring, or risk acceptance is visible.
- Whole Elephant must name the whole object and the result controller before writing
  copy, strategy, or essence claims.
- EDSP must show the two decisive extremes before the practical branch.
- SELA must show cost, throughput, and scale contrast when local excellence is real.
- Anti-Spiral must refuse the third additive rule/fallback before proposing repair.
- AQM may use hypothetical numbers only to expose relationships, not to compute the
  final verdict.

## Boundary

Entry Triage only changes the route, evidence requirement, stopping condition, or first
answer shape. If it changes none of these, skip it.
