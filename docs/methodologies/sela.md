# SELA / 系统效率碾压局部优势

## 这是什么

SELA 是一个顶层战略判断原则，意思是 `System Efficiency over Local Advantage`。

它关心的不是“谁更优秀”，而是“当系统级费效比出现数量级差距时，局部优秀还能不能保住主战场”。

局部优势可以真实存在，也可以非常强，但如果它不能规模化，通常只能退守小众、过渡、特殊或高价值 niche。

## 解决什么问题

SELA 主要处理这类判断：

- 旧范式和新范式如何切换
- 手工卓越和系统效率如何取舍
- 局部控制和规模化系统如何对比
- 真实优势很强，但系统成本曲线更硬时怎么判断长期方向

它不是用来机械宣布“效率一定赢”的。它只是防止人在明明看到系统级压力时，还用局部优秀替自己找永久主战场。

## 核心判断

SELA 先看三件事：

1. 对比是否公平，还是 `best A` 对 `average B`
2. 局部优势是否真的能规模化
3. 旁观者声称的“不耐受”是否只是高估了他人的脆弱性

如果局部优势无法规模化，而系统效率差距在扩大，长期方向通常会向系统方案收敛。

## 怎么用

先判断长期方向，再做时机检查。

长期方向说明“哪条路更可能成为主战场”，时机检查说明“现在该立即切、试点、保留还是等待”。

常见输出是四种：

- `commit`
- `trial`
- `hold`
- `wait`

## 常见误用

- 把 SELA 当成“所有效率都高于一切”的口号
- 把系统趋势直接等同于现在立刻切换
- 把局部优秀误读为长期主战场的保证

## 边界

不要在不可逆伤害、尊严底线、伦理底线或司法/医疗/安全这种场景里把 SELA 当主原则。

这时效率仍然重要，但不能盖过更硬的价值约束。
