# SELA / 系统效率碾压局部优势

## 这是什么

SELA 是 `System Efficiency over Local Advantage`：当系统级费效比开始形成数量级优势时，局部优势即使真实、优秀、令人留恋，也很难继续占据主战场。

它不是一句“效率至上”的口号。SELA 更关心整体与局部之间的真实关系：局部的手艺、经验、审美、控制感可能都是真的，但如果它无法规模化，无法持续吸收反馈，无法把成本曲线打下来，那么长期主战场通常会转向系统方案。

这个方法适合用在战略判断上，尤其是旧范式还很强、新范式还不完美，但系统效率差距已经开始扩大的时候。它提醒人不要把“我能找到一个很强的局部反例”误读成“旧系统还能长期守住整体优势”。

## 解决什么问题

SELA 解决的是一种很常见、也很危险的战略误判：局部优势真实存在，于是人倾向于把它当成长期方向的证明。

典型场景包括：

- 手工卓越与自动化系统的取舍。
- 旧工作流在专家手里表现很好，但新系统在规模、速度、成本上持续逼近。
- 某个团队、个人或局部场景仍然明显领先，但整体生态已经在迁移。
- 用户声称“不可能接受新方案”，但这种不耐受可能只是过渡期错觉。
- 决策者拿 `best A` 对比 `average B`，从而高估旧方案的可守性。

SELA 的价值在于把“局部真的好”和“长期主战场在哪里”拆开。一个东西可以在局部极好，也可以在过渡期非常重要，但这不自动证明它能抵抗系统级效率差距。

## 核心判断

SELA 先看三件事：

1. 对比是否公平，还是拿 `best A` 对 `average B`。
2. 局部优势是否能规模化，还是只能依赖稀缺人力、特殊上下文或不可复制经验。
3. 系统方案的反馈速度、成本曲线和复利能力是否正在扩大差距。

如果局部优势无法规模化，而系统效率差距正在扩大，长期方向通常会向系统方案收敛。局部不一定消失，但更可能退到 niche、过渡期、审美边界、监管边界或高价值例外场景。

SELA 同时要求做时机检查。长期方向正确，不等于现在必须立刻全切。真正可落地的策略通常是可逆、可观察、可修正的持续逼近：先判断方向，再决定是 `commit`、`trial`、`hold` 还是 `wait`。

## 怎么用

使用 SELA 时，不要先问“我喜不喜欢新方案”，而是按顺序问：

1. 旧方案的局部优势是什么？它是真优势、偏好、路径依赖，还是身份认同？
2. 新方案的系统效率来自哪里？速度、成本、分发、反馈、复用、规模还是组合能力？
3. 这个效率差距是否会随时间扩大？扩大速度是否超过旧方案的改良速度？
4. 旧方案能不能把自己的局部优势系统化？如果不能，它会退到哪个边界？
5. 当前行动窗口是什么？现在该全量切换、试点、保留双轨，还是等待关键证据？

输出不必复杂。多数情况下，一句判断加一个动作就够：

- `commit`：方向清楚，风险可控，继续保留旧路径会锁死未来。
- `trial`：方向有压力，但仍需要小规模验证和真实反馈。
- `hold`：旧方案仍承担关键价值，暂时保留，但继续观察系统差距。
- `wait`：证据不足，切换成本高，过早行动会制造不可逆损失。

## 常见误用

最常见的误用，是把 SELA 写成“效率一定赢”。这会把方法变成浅薄口号，也会破坏它真正有用的部分。

另一个误用，是拿长期趋势直接推出短期动作。长期看系统方案更强，不代表今天就应该关闭旧流程、废弃专家判断或牺牲过渡期质量。SELA 应该同时保护方向判断和时机判断。

还有一种误用，是把局部优势贬低成“落后”。这也不对。局部优势往往是系统方案学习的源头，尤其在审美、风险、伦理、复杂上下文和高价值例外场景里，它可能仍然是校准系统的关键输入。

## 边界

SELA 不适合机械套用在不可逆伤害、尊严底线、伦理底线、司法、医疗、安全或监管责任很重的场景里。效率仍然重要，但不能盖过更硬的价值约束。

它也不替代事实研究。系统效率是否真的形成数量级差距，需要证据、数据、案例和运行反馈。SELA 提供的是战略镜头，不是凭空宣布胜负的结论机器。

当问题本身不是长期系统趋势，而是命题结构不清、控制边界不清或任务不可执行时，应转给 `EDSP`、`WAE` 或 `3L5S`。

## 与其他方法的关系

- `using-mindthus` 负责前置校准，避免把流行概念当成真实对象。
- `EDSP` 负责更模糊的结构判断，例如这是不是一个假二选一。
- `WAE` 负责判断这段工作该由流程、agent 判断还是证据控制。
- `tplan` 负责在长任务里记录 SELA 判断、行动窗口和后续证据。
- `Anti-Spiral` 在 SELA 被反复用于证明同一局部立场时提醒暂停。
