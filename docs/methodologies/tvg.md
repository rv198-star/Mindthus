# TVG / Thinking Value-Gain

## 这是什么

TVG 是一个价值加深方法，意思是 `Thinking Value-Gain`。

它不是为了把东西写长，而是为了把已经成型但偏薄的产物，往真正可用的方向加深。

## 解决什么问题

TVG 处理这种情况：

- 文档看起来完整，但下游还得靠人临场补
- 结构严谨，但内容浅
- 很像完成品，但没有足够的判断、证据、取舍和可复用性

## 核心判断

先找最小可冻结模块，再问它还能在哪些地方增加真实价值。

TVG 不问“还能不能再写一轮”，而问“再写一轮会不会真的增加可用性”。

## 怎么用

典型流程：

1. 定义最小模块
2. 找到薄弱的价值面
3. 只在会增加实用价值的地方加深
4. 用 trace 或审计确认不是在为了长度加戏

## 常见误用

- 把深度等同于篇幅
- 把格式完整当成价值完成
- 把 TVG 用到整个项目战略上

## 边界

如果缺的是事实、领域输入、运行时证明或 stakeholder 判断，就别加深，先去补输入。

## 与其他方法的关系

- `3L5S` 可以产出 TVG 的输入模块
- `WAE` 决定这里是流程、判断还是证据问题
- `tplan` 可以承载 TVG 的审计和 trace
