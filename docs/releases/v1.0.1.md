# Mindthus v1.0.1 发布日志

发布日期：2026-06-08

## 版本定位

`v1.0.1` 是 `v1.0 Method Fidelity Framework` 之后的轻量补丁。它不新增方法，也不改变 judge 机制；它新增一个极简使用日志入口，让真实 fidelity judge 结果可以沉淀为后续数据飞轮。

核心变化：

> 让每次真实使用都能留下可脱敏、可校验、可积累的一条记录。

## 这次解决了什么

`v1.0` 已经能校验单次 judge 结果，但单次校验不会自动变成项目资产。`v1.0.1` 补上一个最小习惯：

- 记录场景、方法、模型、baseline 分、constrained 分和约束是否有帮助。
- 没有 baseline 时也允许记录 constrained 分，避免真实使用因为门槛太高而不记录。
- 用 `data/fidelity-usage-log.jsonl` 保存 JSONL，方便后续聚合分析。
- 保留边界：脚本只管记录形状，不判断方法价值。

## 新增

- 新增 `scripts/log-fidelity-usage.py`：追加或校验 Method Fidelity usage log。
- 新增 `data/README.md`：说明默认日志位置和脱敏边界。
- 新增 `docs/internal/fidelity-usage-log.md`：说明 usage log 如何服务 1.0 -> 2.0 的数据飞轮。

## 验证

本补丁应至少通过：

```bash
python3 -m unittest tests.test_v1_0_1_usage_log -v
python3 -m unittest discover -s tests -v
python3 scripts/build-release-pack.py --out /tmp/mindthus-v1.0.1-check --force
```

## 边界

`v1.0.1` 不声明已经具备跨方法的统计结论。它只是让未来的统计结论开始有数据来源。
