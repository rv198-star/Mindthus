# Mindthus v1.0 发布日志

发布日期：2026-06-08

## 版本定位

`v1.0` 是 Mindthus 的第一版正式发布面。它不是再新增一套方法论，而是把
`v0.9 Method Fidelity Harness` 验证过的忠实执行机制，补齐为可发布、可审查、
可打回的执行框架。

核心口径：

> 约束关键判断动作，不约束判断结论。

## 这次解决了什么

### 1. LICENSE blocker closed

本版新增 AGPLv3 `LICENSE`，并通过 `COMMERCIAL-LICENSE.md` 明确商业双授权口径：

- 开源使用、修改、分发和部署按 AGPLv3。
- 闭源商业产品、私有商业平台、商业 SaaS、商业系统集成，或任何不公开对应源代码的
  商业用途，需要单独取得商业授权。

release pack 会把 `LICENSE` 和 `COMMERCIAL-LICENSE.md` 同步带入 Claude Code、
Codex 和 OpenCode 三个平台包。

### 2. Judge automation blocker closed

新增 `scripts/run-fidelity-judge.py`：

- 可从方法输出生成可复现 judge prompt。
- 可校验 judge JSON 是否包含完整维度评分、证据摘录、总分和元规则确认。
- 只检查 judge 记录形状和覆盖，不替代 human / LLM judge 的语义判断。

脚本输出仍保留边界提醒：judge automation does not validate strategic truth。

### 3. challenge_premise escape blocker closed

SELA judge rubric 和 judge runner 增加 escape review：

- 当输出使用 `not_applicable`、`transfer` 或 `challenge_premise` 时，judge 不能自动放行。
- judge 必须审查这个退出本身是否成立。
- 自动化记录必须包含 `escape_review`，说明退出类型、是否成立、理由和 reviewer action。

这避免 fidelity harness 从“逼真判断”被绕成“逼一个合理的不判断理由”。

### 4. Cross-model baseline blocker closed

新增 `tests/sela/cross_model_baseline_2026-06-08.md`，在既有 v0.9 SELA packet 之外，
补跑 `opencode/deepseek-v4-flash-free` 的 baseline-vs-constrained 小样本。

结论保持克制：

- constrained prompt 在两个已测模型记录中稳定改善 SELA 忠实执行。
- 最稳定的改善点是 fair comparison、长期趋势 vs 立即行动分离、misuse challenge。
- 这足以关闭单模型-only blocker，但不声明所有模型、所有方法的普适鲁棒性。

## 验证结果

本版验收命令：

- `python3 -m unittest discover -s tests -v`
- `python3 -m unittest discover -s tests/tplan -v`
- `python3 -m py_compile $(git ls-files 'skills/**/*.py' 'scripts/*.py' 'tests/*.py' 'tests/tplan/*.py') scripts/run-fidelity-judge.py tests/test_v1_0_readiness.py`
- `git diff --check`
- `python3 scripts/build-release-pack.py --out /tmp/mindthus-v1-release-audit --force`

验收目标：

- 主测试全绿。
- tplan 回归全绿。
- release pack 三平台包含 `LICENSE`、`COMMERCIAL-LICENSE.md`、judge 脚本和 SELA rubric。
- README、CHANGELOG 和 release pack metadata 均进入 `v1.0` 口径。

## 边界

`v1.0` 可以声明 Method Fidelity Framework 进入正式发布面，但仍不声明：

- 自动 judge 能判断战略结论真伪。
- SELA 小样本能代表所有模型和所有方法。
- 形状校验通过等于方法判断正确。
- Mindthus 能替代领域事实、法律、医疗、安全或组织权限判断。
