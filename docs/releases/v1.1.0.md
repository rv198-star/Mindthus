# Mindthus v1.1.0 发布日志

发布日期：2026-06-11

## 版本定位

`v1.1.0` 是一次 agent 使用体验增强版本。它不改变 Mindthus 的基本定位：Mindthus 仍然是给 agent 用的判断和执行基础设施，而不是给人填表的流程框架。

这次主要补强两件事：

- TVG 不只能按默认通用标准让文本更有判断、更可交接，也可以按任务自定义价值锚点，让文本更接近某个明确的“好”。
- TPlan 不只让单个子任务记录自己的问题，也可以把共享阻塞和风险提升到 Mission 级别，影响后续判断。

## 新增

### TVG Value Profile

TVG 现在可以支持自定义价值锚点：这个任务里什么算好、什么算跑偏、哪些约束不能破、优先增强哪类价值。用户可以让 TVG 沿着自己定义的好标准迭代优化，而不是只能使用默认通用实用价值。

这个能力适合两类场景：

- 普通文档、PRD、方案、方法稿：继续使用默认 profile，让 TVG 保持“更有判断、更有证据边界、更可交接”的通用目标。
- 有明确领域标准的输出：提供自定义 profile，让 TVG 知道这次真正要增强的价值是什么。

本版随包包含默认通用价值 profile，也包含两个影视提示词 / 分镜方向的示范 profile：

- 邵氏清水湾棚拍时代武侠 / 神怪：用于展示怎样把一个有明确审美目标的 profile，写成可审查的价值定义、输出形态和扩写策略。
- 胡金铨武侠电影：用于展示怎样把相近但不同的风格边界写清楚，尤其是避免把邵氏清水湾棚拍美学误当成胡金铨武侠语法。

这两个 profile 是示范用例，不是电影史分类结论，也不是对所有图像模型的风格稳定性保证。它们用于展示如何编写和使用 TVG profile：定义“什么算好”、输出厚度落在哪些可审查单位上，以及怎样把剧本文字扩成可审查的多镜头提示词。

### TPlan Shared Risk Context

TPlan 现在允许子任务共享阻塞项和风险。某个子任务发现的阻塞、环境问题、证据风险或共享依赖风险，可以提升到 Mission 级别，让其他子任务和后续决策也能看到。

对用户来说，这意味着 agent 在评估下一步是否继续、切换路径、健康检查、暂停或收束时，不只看当前子任务自己的局部状态，也会综合已经发现的共享阻塞和风险，判断任务可行性和综合价值。

## 修复

- TPlan continuation authorization：修复晚期小缺陷、证据形态缺陷或局部未收敛时，agent 沿同一路径惯性续跑的问题。现在继续前必须说明为什么这条路径仍然值得继续，以及授权依据是什么。
- Codex 安装路径：`scripts/install-skills.sh codex` 现在默认安装到 `${CODEX_HOME:-~/.codex}/skills/mindthus`，不再把 Codex skills 链到 `.agents` 路径。
- Codex App 技能发现：修复 `tplan` 在 Codex CLI 可见、但 Codex App 看不到的问题。入口 frontmatter 现在保持严格 YAML 兼容，Mindthus 各个 `SKILL.md` 入口也都控制在 8KiB 以内，降低 App 侧发现失败的风险。
- Release pack 边界：安装包构建现在会过滤测试、日志、运行状态、生成图片/视频和临时产物，避免把开发过程里的记录混进发布包。包内只保留必要的 skill、公开方法文档、安装脚本和 `tplan/templates/evidence.jsonl` 模板。

## 验证

发布前应至少通过：

```bash
python3 -m unittest discover -s tests
python3 scripts/build-release-pack.py --out /tmp/mindthus-v1.1.0-check --force
```

本次重发还补跑了 release pack 审计：确认三平台安装包不包含 `tests/`、`logs/`、`.tplan/`、`.tvg/`、运行产物、图片/视频或临时文件；唯一保留的 `jsonl` 是 `tplan/templates/evidence.jsonl` 模板。

## 边界

`v1.1.0` 不声明 TVG profile 能自动证明某个领域审美正确，也不声明示范 profile 已经覆盖所有图像模型的视觉漂移问题。它也不声明 TPlan 可以自动替用户决定是否继续。它提供的是更清楚的价值锚点、更好的共享风险可见性，以及更稳的继续判断约束。
