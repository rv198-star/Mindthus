# Mindthus v1.3.0 发布日志

发布日期：2026-06-27

## 版本定位

`v1.3.0` 是一次 router-boundary + plugin-startup release。它不新增新的主方法，而是把最近三条相关变化收束成一个更干净的对外版本面：

- `#66` 让 Claude Code 和 Codex plugin 在 startup context 有轻量 Activation Router。
- `#64` 把 WAE 收窄回 agentic-system control-boundary。
- `#65` 把 TVG exit audit 收回 TVG loop 内部，并同步修正 tplan 的相关术语边界。

这更像一次 minor release，而不是简单 patch，因为它改变了 plugin startup 提示面和方法路由边界的公开语义。

## 安装入口

普通用户继续按 release asset 安装；plugin mode 下载 plugins 包，skills-pack mode 下载 skills 包：

```bash
VERSION=1.3.0
curl -L \
  -o /tmp/mindthus-plugins-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-plugins-${VERSION}.tar.gz"
rm -rf /tmp/mindthus-plugins
mkdir -p /tmp/mindthus-plugins
tar -xzf /tmp/mindthus-plugins-${VERSION}.tar.gz -C /tmp/mindthus-plugins --strip-components=1

curl -L \
  -o /tmp/mindthus-skills-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-skills-${VERSION}.tar.gz"
rm -rf /tmp/mindthus-skills
mkdir -p /tmp/mindthus-skills
tar -xzf /tmp/mindthus-skills-${VERSION}.tar.gz -C /tmp/mindthus-skills --strip-components=1
```

本次 release asset 名称是 `mindthus-plugins-1.3.0.tar.gz` 和 `mindthus-skills-1.3.0.tar.gz`。

## 这次解决了什么

### #66 Activation Router

Claude Code plugin 现在会生成真实 `SessionStart` hook，把一个小型 startup router context 注入 session。Codex plugin 则继续使用当前可用的 `interface.defaultPrompt`，保持在 loader 的 128-byte 限制内。

这一层只做轻量唤醒，不把 Mindthus 变成强制流程：

- hard judgment point 才优先考虑 `mindthus:using-mindthus`
- 简单、明确、低风险任务直接做
- 缺事实、缺文件、缺运行时证据时先补证据
- 如果上游已有 Superpowers Brainstorm 等 discovery workflow，Mindthus 不重复那层展开

### #64 WAE 路由修复

WAE 现在明确是 agentic-system control-boundary lens，而不是泛化的“边界方法”。

短规则就是：

```text
No agentic system, no WAE.
No controller mismatch, no WAE.
```

这意味着普通概念边界、组织责任、产品分类、issue 粒度或结构判断，不应该因为出现 process / evidence / review 词汇就被错误路由到 WAE。

### #65 TVG 边界修复

TVG 的 audit 现在明确只属于 TVG loop 内部退出判断：

```text
TVG audit = TVG-loop exit judgment.
No active TVG loop, no TVG audit.
```

它不再因为用户说了 audit / review / check 就外部化为通用审计路线。release、code、workflow、factual、method、strategy 和 requirement-boundary 审计，需要按对象路由。

与此同步，tplan 语义也被钉住：

- `artifact_value_gain` 是 decision hook
- 它不是 Mission Pulse `next_gate`
- TVG 内部动作词不会直接进入 Mission mutation recommendation

## 边界

- 本版不 vendoring Superpowers Brainstorm，也不占用 Superpowers 命名空间。
- 本版不把 Mindthus 变成 mandatory startup workflow。
- 本版不把 TVG 重新定义为外部 release / code / workflow audit 方法。
- 本版不把 WAE 扩成普通治理、责任归属或分类边界方法。
- 本版不宣称 `#50` 已完成真实 replay 级 wake-up lift 认证；`#50` 仍是后续行为认证 issue。
