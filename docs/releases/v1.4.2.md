# Mindthus v1.4.2 发布日志

发布日期：2026-07-06

## 版本定位

`v1.4.2` 是一次判断权与认知原语架构收束 release。

它继续处理 v1.4.x 的核心问题：防止局部真相冒充整体。区别在于，本版不再靠继续堆字段修失败，而是把 Whole Elephant 压回少而硬的主线，同时把容易冲突的横切原语拆清楚：谁主导首句判断，谁只提供约束、证据或路由支持。

本版重点补强四件事：

- `Whole Elephant Protocol / 全象流程`：以 `canonical_object / result_controller / misdirection_if_local_wins` 三根硬支柱锁定完整对象、结果主控和优化偏移。
- `Decision Context Calibration / 决策语境校准`：处境化判断先看当前决策者、时点、目标函数和可接受损耗。
- `Aspect Ownership Matrix / 切面主导权矩阵`：多个横切原语同时触发时，不把输出平均成温和但无判断力的答案。
- `MPG Scalar Commitment Unpack / MPG 标量承诺显影`：把压缩的单点承诺显影成 MPG 能接住的主线-载体-路径-暴露-承诺结构。

## 安装入口

普通用户继续按 release asset 安装；plugin mode 下载 plugins 包，skills-pack mode 下载 skills 包：

```bash
VERSION=1.4.2
curl -L \
  -o /tmp/mindthus-plugins-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-plugins-${VERSION}.tar.gz"
rm -rf /tmp/mindthus-plugins
mkdir -p /tmp/mindthus-plugins
tar -xzf /tmp/mindthus-plugins-${VERSION}.tar.gz -C /tmp/mindthus-plugins --strip-components=1

curl -L \
  -o /tmp/mindthus-skills-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-skills-${VERSION}.tar.gz"
rm -rf /tmp/mindthus-skills
mkdir -p /tmp/mindthus-skills
tar -xzf /tmp/mindthus-skills-${VERSION}.tar.gz -C /tmp/mindthus-skills --strip-components=1
```

本次 release asset 名称是 `mindthus-plugins-1.4.2.tar.gz` 和 `mindthus-skills-1.4.2.tar.gz`。

## 这次解决了什么

### 把 Whole Elephant 从字段中心拉回判断主线

v1.4.1 后的外部审计指出：Whole Elephant guardrail 有变成新 judgment center 的风险。字段越来越多，反而会稀释主判断注意力。

v1.4.2 把核心收束成 `Compact Semantic Triad / 三根硬支柱`：

```text
canonical_object
result_controller
misdirection_if_local_wins
```

意思很简单：先锁定完整对象，再看谁控制目标结果，最后说明如果局部真相赢得定义权，会把优化方向带偏到哪里。

### 用后果对比抵抗局部真相反复施压

本版加入 `Contrastive Consequence Probe / 后果对比探针`：

```text
local_frame_wins
whole_object_wins
better_direction_for_target
```

它不是新方法，也不是完整 EDSP。它只帮助 agent 看见：如果接受局部框架，下一步会优化什么；如果保留完整对象，下一步会优化什么；哪条路更不容易优化错方向。

### 让处境化判断不再被抽象公平稀释

有些问题不是纯客观定义题，而是带着决策者、时点、目标和损耗边界的问题。比如一个 27 英寸 4K/5K/BetterDisplay 讨论中，抽象地说“各有道理”可能滴水不漏，但不一定服务当前楼主的真实决策。

`Decision Context Calibration` 要求先判断：

- 谁在做决定；
- 当前是在购买前、使用中、排障中还是发布前；
- 目标函数是什么；
- 哪些代价可以接受；
- 哪个框架对当前决策更有定义权。

### 防止多个横切原语合成无锐度答案

`Aspect Ownership Matrix` 明确：多个原语可以同时激活，但首句判断只能有一个 `judgment_owner`。其他原语可以做约束、支持、证据上限或表达修正，但不能把回答冲淡成“每边都有道理”。

这不是鼓励偏激，而是避免 AI 用表面公平逃避判断。

### 改善 MPG 的入口形状

MPG 需要 `mainline / carrier / path_volatility / exposure / commitment`，但用户常常只问一个压缩问题：买不买、卖不卖、留不留、要不要继续。

`MPG Scalar Commitment Unpack` 只做一件事：当单点承诺背后藏着主线承载结构时，先把结构显影出来，再决定是否交给 MPG。它不替 MPG 下结论，也不把简单问题拖进方法。

### 拆薄 shared-primitives 入口

`docs/methodologies/shared-primitives.md` 已从规则仓库改成索引和 cross-primitive map。高复杂原语拆到：

- `docs/methodologies/primitives/aspect-ownership.md`
- `docs/methodologies/primitives/decision-context-calibration.md`
- `docs/methodologies/primitives/frame-fitness-check.md`
- `docs/methodologies/primitives/mpg-scalar-commitment-unpack.md`
- `docs/methodologies/primitives/whole-elephant-protocol.md`
- `docs/methodologies/primitives/expression-pressure-and-gates.md`

这一步不是 #83 的完成版。#83 继续跟踪剩余小原语的全量拆分。

## 工程变化

- 新增 `scripts/primitives/whole_elephant_validator.py`，让独立 validator 与 `using-mindthus` 集成 validator 共用同一套核心逻辑。
- `scripts/primitives/validate_whole_elephant.py` 退回 CLI wrapper，减少 validator drift。
- release pack 会携带 split primitive docs、calibration pairs、primitive checker 和 shared Whole Elephant validator。
- `scripts/log-mindthus-runtime.py --strict` 现在会检查 tracked files 是否存在，不只检查 marker 和 hash。

## 边界

- 本版不新增独立 skill。
- 本版不把 Whole Elephant 变成默认 route。
- 本版不关闭 #83；小原语全拆仍是后续文档架构任务。
- `MPG Scalar Commitment Unpack` 是 support primitive，不是新的 MPG 替代品。
- `Decision Context Calibration` 不抹掉事实；事实仍约束答案，当前决策语境只决定哪些事实有判断权。
- 这些脚本和 validator 仍不决定语义真理，只阻断已知高风险形状和 drift。
