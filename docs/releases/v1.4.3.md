# Mindthus v1.4.3 发布日志

发布日期：2026-07-06

## 版本定位

`v1.4.3` 替代并吸收已撤回的 v1.4.2。

v1.4.2 GitHub Release 已删除，不再作为安装入口；v1.4.2 的架构内容保留在本版 changelog 与发布说明中，并由 v1.4.3 统一承载。

它不试图关闭 #85，而是先把一个容易长期抖动的边界钉住：**读取某个方法作为参考或验收 rubric，不等于当前任务由这个方法主导。**

这次修复的目标保持克制：

- 防止 “确认某会话是否启用了 MPG-AQM” 被表达成当前任务激活 MPG；
- 防止普通 release/code/log review 被 `review / audit / check` 词面误拉进 TVG；
- 保持入口层轻，不用继续叠补丁扩大方法中心。

## 吸收自 v1.4.2 的内容

v1.4.2 的核心能力不丢弃，只是不再作为独立 release 安装面存在：

- `Whole Elephant Protocol / 全象流程` 收束为 `Compact Semantic Triad / 三根硬支柱`：`canonical_object / result_controller / misdirection_if_local_wins`。
- `Contrastive Consequence Probe / 后果对比探针` 比较 `local_frame_wins` 与 `whole_object_wins`，让 agent 看见不同定义权会导向什么优化方向。
- `Decision Context Calibration / 决策语境校准` 用决策者、时点、目标函数和可接受损耗校准处境化判断。
- `Aspect Ownership Matrix / 切面主导权矩阵` 防止多个横切原语平均成“各打一板”的无锐度输出。
- `MPG Scalar Commitment Unpack / MPG 标量承诺显影` 把压缩承诺显影为 `mainline / carrier / path_volatility / exposure / commitment`，作为 MPG 的前置路由支持。
- `shared-primitives.md` 拆成轻索引，高复杂原语细节移入 `docs/methodologies/primitives/`。
- Whole Elephant 独立 validator 与 `using-mindthus` 集成 validator 共用 `scripts/primitives/whole_elephant_validator.py`，减少 drift。

## 主要变化

### 方法引用边界

`using-mindthus` 增加 `Method Reference Boundary / 方法引用边界`：

```text
method name in an inspection request is evidence scope, not route ownership
```

当用户问的是会话取证、日志审计、方法使用核验时，agent 可以读取 MPG-AQM、TVG 或其他方法规则作为对照标准，但不能把“读了规则”说成“当前任务由该方法主导”。

### 新增负向测试

新增压力场景：

```text
019f359a-6aa2-78c0-9ac5-822abae99495，这个会话我不确定它是否有启用 MPG-AQM，帮我确认下。
```

期望行为：

- 先识别为 `session forensics / log audit`；
- 可以用 MPG-AQM 规则作 rubric；
- 检查目标会话证据；
- 明确区分目标会话是否用了 MPG-AQM，和当前确认请求是否由 MPG 主导。

### TVG 外部审查防回归

本轮实测没有复现 TVG 被 release readiness 审查误触发，但保留负测作为长期观察：

- release readiness 审查按测试、风险、阻塞和证据走；
- code review 按代码对象走；
- log/session audit 按取证对象走；
- TVG 只在 active TVG loop + bounded artifact value-gain target 存在时进入。

### 入口体积控制

- `skills/mpg/SKILL.md` 已压回 10KiB 以下，完整方法论保留在 `resources/methodology.md`。
- `using-mindthus` 作为路由入口，预算调整为 11KiB / 1100 words。它仍必须保持薄入口，但不为了硬压 10KiB 删除关键路由锚点。

## 边界

- v1.4.2 已撤回；不要从 v1.4.2 安装或引用其 release asset。
- 本版不关闭 #85；该 issue 继续观察真实会话里的方法引用/激活边界。
- 本版不新增独立 skill。
- 本版不改变 MPG、SELA、TVG 的正常适用范围。
- 本版不声称解决所有 discovery 层误触；显式提到某技能名时，外层 skill discovery 仍可能读取该 skill，关键是最终不得让它错误主导当前任务。
