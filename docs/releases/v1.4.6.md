# Mindthus v1.4.6 发布日志

发布日期：2026-07-14

## 版本定位

`v1.4.6` 把前一轮已经合入 main、但尚未进入正式 release pack 的 TVG 高级 Profile
案例补进发布版本，同时修复 Codex 插件图标在黑 / 白色主题下的对比度问题。

这版的重点不是再增加一条风格化示例，而是把“电影级出图”作为一个完整的 TVG-Profile
高级用法交付：Profile 定义价值标准，realization surface 定义可审查的镜头单位，
gain policy 定义值得加深的动作，runtime support 提供确定性辅助，但不替 TVG 做语义退出判断。

## TVG-Profile：电影级出图高级包

`docs/methodologies/tvg.md` 现在把 `cinematic-colossal-realism` 作为四层高级包的明确入口。
它服务于电影级图像 prompt packet 和生成图 review，重点是把“cinematic”翻译成镜头、
光线、尺度、空间关系、物理材质和第一读压力，而不是继续堆 cinematic 形容词。

这个案例与邵氏 / 胡金铨两个风格型影视 Profile 不同：它不是两个风格化画面的并列示范，
也不是导演风格分类结论。它展示的是如何把外部 cinematic skill 拆成 TVG 可审查的四层
Profile，并区分 single-pass profile power 与 loop-assisted production 的证据边界。

随包内容包括：

- `profile.md`：四层 Profile、硬性 veto、pressure guidance 与 image review gate；
- `resources/`：camera / lighting、field template、subject taxonomy、scene defaults、
  image audit rubric 和 negative constraints；
- `scripts/`：subject classification、prompt skeleton、field lock validator 和 lint；
- `examples/`：single-pass profile power、loop-assisted image comparison、research log 与 migration notes。

脚本只报告结构性 findings，不决定美学是否成功、Profile 是否成熟，也不能替 TVG 决定
继续还是退出。单次 prompt 或图像 smoke 只能支持“该 Profile 对这类输出有可用约束力”，
不能证明所有模型都会稳定生成对应结果。

## Codex 插件图标主题适配

此前 release pack 的 `mindthus-icon.png` 是透明底黑线，深色主题下会失去对比度；Logo 也
只有固定的浅色版本。v1.4.6 做了两层修复：

- `composerIcon` 改为带自包含浅色底的 `assets/mindthus-icon.svg`，浅色和深色主题都保持可见；
- `logo` 使用 `assets/mindthus-logo.svg`，并增加官方支持的 `interface.logoDark`，指向
  `assets/mindthus-logo-dark.svg`。

这样即使宿主只识别单一 composer icon，图标也不会在深色背景上消失；支持 `logoDark` 的
Codex 界面则会按主题切换 Logo。

## 继承的发布边界

- 显式调用 `using-mindthus` 或具体 skill 是可靠主路径；host 自然唤起仍是 best-effort。
- `using-mindthus` 的轻量入口和五种发布布局的条件资源保持 v1.4.5 的边界。
- 公开 50 场景研究仍是可复查测量，不构成自动唤起认证；`1.447 < 1.5` 的结果不被包装成通过。
- `v1.4.4-diag`、register-hints 和 semantic triage 仍是诊断历史，不进入本版产品承诺。
- release pack 不包含 tests、logs、内部研究文档或 benchmark 运行产物。

## 安装

plugin mode 适用于 Codex App / CLI 与 Claude Code：

```bash
VERSION=1.4.6
curl -L \
  -o /tmp/mindthus-plugins-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-plugins-${VERSION}.tar.gz"
```

skills-pack mode 适用于 OpenCode 或手动复制 skills：

```bash
VERSION=1.4.6
curl -L \
  -o /tmp/mindthus-skills-${VERSION}.tar.gz \
  "https://github.com/rv198-star/Mindthus/releases/download/v${VERSION}/mindthus-skills-${VERSION}.tar.gz"
```

最终 Release 资产同时提供 `SHA256SUMS`，它由两个 tarball 生成后单独计算。

## 升级影响

- 无 schema migration。
- 无新增必选依赖。
- 已安装用户可直接用 v1.4.6 覆盖旧 plugin / skills 包。
- 依赖自动 discovery 的工作流仍应把它视为 best-effort；关键判断请显式调用。
