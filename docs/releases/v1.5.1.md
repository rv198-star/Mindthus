# Mindthus v1.5.1 发布说明

准备日期：2026-07-19

状态：`v1.5.1` GitHub Release 已发布，提供 Stable plugins、Stable skills 和 Codex ROI Beta
experimental asset；不发布 marketplace。

## 版本定位

`v1.5.1` 是 1.x Stable 线的可靠性 patch。它修复 v1.5.0 审计发现的三个问题，
不新增方法论，不改变 `using-mindthus` 的调用边界，也不把 ROI Beta 的有损唤起合同带入 Stable。

## #125：TPlan 状态与成本一致性

- lifecycle journal 在 Mission 状态落盘失败后可恢复，避免 trace、evidence 与 Mission 状态分叉。
- trace append 与 journal recovery 使用同一执行锁，防止并发追加被恢复过程覆盖。
- 成本树在完整 span 图上选择 leaf-most usage owner，避免跨 attribution projection、嵌套
  agent turn 和 partial usage 的重复计费。
- durable sync 只覆盖恢复所需的 lifecycle journal；高频普通事件不承担无意义的完整同步开销。

## #127：发布包 helper 自定位

- 每个打包后的 Python helper 都从自身安装位置定位共享 `_runtime`。
- 正确 runtime 根目录即使已经出现在 `sys.path` 后部，也会被提升到首位，避免同名外部包遮蔽。
- 回归测试覆盖 Codex plugin、Codex skills、Claude plugin、Claude skills 与 OpenCode 布局，
  并显式覆盖 macOS `/var` 到 `/private/var` 的路径别名。

## #126：TVG Atlas 来源绑定

- label manifest 记录源图 SHA-256、尺寸、region 顺序和边界；selection board 在裁切前校验全部绑定。
- 哈希和图像解码使用同一份内存快照，消除检查与使用之间的 TOCTOU 窗口。
- trace contract 升级到 v2，补齐 direct-delivery 状态及 `evidence_class`。
- 随仓库提供的 PPM 仅是 deterministic structural fixture；校验器强制它声明没有模型调用，
  且不能作为审美、模型生成或 Profile 成熟度证据。

## 独立审计

三个问题均由与实现责任分离的只读审计者复核。首轮发现的问题全部回到实现和测试修复，
最终 #125、#127、#126 均得到 PASS。审计结论不替代本地测试和构建证据。

## 验证

发布构建必须完成：

```bash
python3 -m unittest discover -s tests -p 'test_*.py'
python3 scripts/build-release-pack.py --package plugins --out /tmp/mindthus-1.5.1-plugins --force
python3 scripts/build-release-pack.py --package skills --out /tmp/mindthus-1.5.1-skills --force
```

还必须核对 Stable plugins、Stable skills、ROI Beta 三个包的 manifest 版本、`SHA256SUMS`、
隔离安装和回滚路径。GitHub CI 因费用停止，本版使用本地可复现验证，不伪造或绕过不存在的
CI 成功状态。

## 发布边界

- `v1.5.1` GitHub Release 同时提供 `mindthus-plugins-1.5.1.tar.gz`、
  `mindthus-skills-1.5.1.tar.gz`、`mindthus-beta-1.5.1-roi-beta.tar.gz` 与 `SHA256SUMS`。
- ROI Beta 仅是 Codex 的 experimental asset，不是第二个 GitHub Release，也不发布 marketplace。
- README 将 `v1.5.1` 作为当前 Stable 下载入口；需要完整、保守行为时默认选择 Stable。
