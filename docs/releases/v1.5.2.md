# Mindthus v1.5.2 发布说明

发布日期：2026-07-22

状态：`v1.5.2` GitHub Release 已发布，提供 Stable plugins、Stable skills 和 Codex ROI Beta
experimental asset；不发布 marketplace。

## 版本定位

`v1.5.2` 是 1.x Stable 线的可靠性 patch。它收敛了 LoopX 对比后确认有实际收益的三项
TPlan 修复：Mission 不应被中途交互劫持，长时间 no-op 不应让用户误以为任务死锁，执行结果
也不能被未验证遥测或写回伪造。它不新增方法论，不要求迁移既有 Mission。

## #134：Bounded Interaction Guard

- 既有 Mission 的 continuation boundary 不再因一条插入消息而默认转交控制权。
- 能提供中断守卫的 host 使用 hook 进行强制边界控制；不支持 hook 的 host 明确降级为
  prompt 合同与 safe-stop 协议，不声称具备同等的 mutation prevention。
- safe-stop、operator resume 与 completion 都有可验证的状态边界，避免一边回复插话、一边
  静默丢失原目标。

## #135：Quiet no-op heartbeat

- 连续两次没有状态变化时保持安静，避免状态轮询变成刷屏。
- 从第三次开始给出低频间断反馈，帮助用户区分“仍在运行”与“可能卡住”，但不把 heartbeat
  当作实际进展或成功证据。

## #136：Validated Outcome Attribution

- 观测到的遥测先被归属和校验，再决定是否能写回 Mission outcome。
- 重复事件、错误 owner、无效证据或未完成 writeback 都不能伪造已完成、已交付或成本归属。
- outcome report 的新增字段保持兼容；证据不足时明确保留未知或拒绝写回。

## 验证

发布构建完成以下本地可复现验证：

```bash
python3 -m unittest discover -s tests -p 'test_*.py'
python3 scripts/build-release-pack.py --package plugins --out /tmp/mindthus-1.5.2-plugins --force
python3 scripts/build-release-pack.py --package skills --out /tmp/mindthus-1.5.2-skills --force
```

并核对 Stable plugins、Stable skills、ROI Beta 三个包的 manifest 版本、`SHA256SUMS`、
归档内容与隔离安装路径。GitHub CI 因费用停止，本版使用本地可复现验证，不伪造或绕过
不存在的 CI 成功状态。
