# Mindthus v2.0.0-beta.1 预发布说明

预发布日期：2026-07-19

## 定位

这是面向高能力 Codex 的 Efficiency-first Beta。它用更薄的 `using-mindthus` 降低常驻
方法内容带来的加载与输入成本，并允许“不改变行动”的被动唤起损失。它不替代 1.x Stable，
也不声称已经覆盖所有 owner、认知原语或真实任务。

## 已有行为证据

冻结的 ROI.2 资格结果表明，在两个 Anti-Spiral 边界任务中，候选相对 Stable 的中位数：

- 显式加载 Mindthus bytes 下降 42.17%；
- host-reported uncached input tokens 下降 53.63%；
- wall duration 下降 10.07%。

更广的先前样本显示更高降幅，但包含“未唤起且行动未退化”的场景，不能解释成完整 recall
保持。所有结果来自有限样本的 `gpt-5.6-sol / xhigh` 运行，不构成普遍可靠率证明。

## #128 装配修复

- Shared core 固定为 1.5.1 repaired commit `4f2dbc756625dfde5f4dad7dfcb2c04730f0217b`。
- 历史 convergence commit 只保留为决策证据标识，不再是构建依赖。
- builder 只接受 clean checkout，并从当前不可变 Git commit 读取 profile、register、overlay 与历史合同输入。
- artifact 内所有 `mindthus:` 引用改写为 `mindthus-beta:`；诊断脚本只报告 Beta 版本、marketplace 与 cache 坐标。
- packaged profile 记录全部装配输入、最终 plugin manifest 和 capability register 的 SHA-256。
- capability register 使用 artifact-local 路径；builder 可生成固定时间、owner、顺序和权限的可复现 tar.gz。

## 安装隔离

Beta package identity、marketplace identity、cache coordinates 和 skill namespace 均为
`mindthus-beta`。Stable 与 Beta 可以分别安装和移除；不会自动迁移用户现有 Stable。

回滚目标为保留的 `v1.4.6` Stable。发布资产只面向 Codex，不包含 Claude、OpenCode 或
marketplace 自动部署。

## 已知边界

- ROI.2 是有损优化：部分被动原语唤起率可能低于 Stable。
- 现有资格证据不能证明所有七个 owner 或所有真实任务无 decision-changing regression。
- 不得因为本次装配审计通过，就把 Beta 升格为正式 2.0。
- 新的 owner 局部修正仍必须由新的 decision-changing 失败证据授权。

## 资产验证

- `mindthus-beta-2.0.0-beta.1.tar.gz`
- `SHA256SUMS`

GitHub Release 必须设置为 prerelease，不得设置为 latest Stable，也不发布 marketplace。
