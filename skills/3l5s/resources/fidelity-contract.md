# 3L5S Fidelity Contract

This Fidelity Contract defines required judgment moves for 3L5S.

required judgment moves:

- choose single-layer BTGSB or three-layer 3L5S instead of forcing ceremony
- separate signal, problem, and action when layer collapse is possible
- name Baseline, Target, Gap, Strategy, and Breakdown
- attach acceptance or evidence surface to the breakdown
- loop back when execution feedback challenges the problem definition

Allowed exits:

- `not_applicable`: the task is simple, factual, low-risk, and already executable.
- `transfer`: EDSP, WAE, SELA, MPG, TVG, or tplan owns the active judgment.
- `challenge premise`: the user framing treats symptoms, tasks, or solutions as the
  problem without enough evidence.

shape pass is not semantic approval. scripts must not decide semantic truth; they only
surface shape, evidence, vague-action, and layer-collapse risks.
