# EDSP Fidelity Contract

This Fidelity Contract defines required judgment moves for EDSP.

required judgment moves:

- decide whether uncertainty is structural or scenario-specific
- push key variables to meaningful extremes
- collapse outcomes into a structural coordinate system
- run Builder / Challenger / Synthesizer pressure
- stop before Scenario Projection if the coordinate system is unstable
- name evidence or boundary conditions that could overturn the structure

Allowed exits:

- `not_applicable`: the missing input is evidence, domain research, runtime proof, or
  stakeholder authority.
- `transfer`: 3L5S owns problem definition, SELA owns system/local trend judgment, MPG
  owns path-carrying, or WAE owns control boundaries.
- `challenge premise`: the binary choice or proposition is malformed.

shape pass is not semantic approval. scripts must not decide semantic truth; a clean
coordinate system can still be wrong.
