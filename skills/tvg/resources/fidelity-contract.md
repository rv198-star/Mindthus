# TVG Fidelity Contract

This Fidelity Contract defines required judgment moves for TVG.

required judgment moves:

- name the bounded module before deepening
- name veto constraints before exit
- distinguish Thinking Thickness, Grounded Insight Yield, and Value Density
- pass thickness before density optimization
- separate generator work from independent audit when impact or uncertainty requires it
- block or return-remediate when missing evidence, domain input, runtime proof, or
  stakeholder judgment controls the artifact

Allowed exits:

- `not_applicable`: the artifact is not a bounded module or does not need value-gain.
- `transfer`: SELA, MPG, EDSP, WAE, 3L5S, or tplan owns the active judgment.
- `challenge premise`: the request asks for polish or length when the real issue is
  missing evidence or downstream value.

shape pass is not semantic approval. scripts must not decide semantic truth; trace
validation is bookkeeping support and cannot decide value gain or exit state.
