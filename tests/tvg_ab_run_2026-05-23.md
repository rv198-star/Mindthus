# TVG v0.2 -> v0.3 A/B Run Record — 2026-05-23

This record captures the A/B checks for commit `545d004` plus the follow-up test
artifacts added after that commit.

## Change Under Test

TVG v0.3 added two intended behavior changes:

1. `veto_constraints`: unacceptable states are explicit exit blockers, not ordinary
   value-gain axes.
2. `independent_auditor`: high-impact, high-uncertainty, or handoff-critical modules
   should separate generator work from the exit auditor.

## Overall Judgment

The v0.3 change is not primarily about forcing more `blocked` outcomes. Across four
scenarios, the main difference is making two previously implicit controls explicit and
auditable:

- unacceptable states must be named and checked as veto constraints;
- high-impact/handoff-critical exits should not rely on generator self-audit.

The A/B result supports keeping the change. Scenario 1 also shows why future tests
should not rely only on exit state: an old method may reach the same `blocked` result
while still lacking the new audit fields that make the reason reviewable.

Scenario 3 adds a second veto-constraint pressure where both variants repair the
artifact but only v0.3 records the veto check. Scenario 4 adds a stronger independent
auditor pressure, while also exposing a runner limitation: clean live v0.2 baselines
require isolating global skill discovery, not just changing the working directory.
